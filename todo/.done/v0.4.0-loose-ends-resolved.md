# v0.4.0 loose ends (resolved part, split from v0.4.0-loose-ends.md)

Loose ends identified during the v0.4.0 redesign session. This file holds the items completed since; the remaining items stay in `todo/v0.4.0-loose-ends.md`.

### Light-theme accent-hi collision

After the AA palette fix darkened `--accent` from `#2d7ff9` to `#2d6cf4`, the light theme's `--accent-hi` (`#1f6ce0`) is nearly indistinguishable from `--accent` (contrast ratio 1.03:1 between them). Adjust `--accent-hi` in the light theme block of `tokens.css` to restore visible differentiation. The contrast test in `tests/test_contrast.py` will verify the new value still passes AA.

Affected files: `assets/css/tokens.css`
Effort: 5 minutes

### E2e test flakiness

Two Playwright tests showed intermittent failures during the session:

- `forms.spec.js:37` ("form submission includes checkbox and radio values") — passes on re-run
- `tooltip-hovercard.spec.js` — fixed from `toBeVisible` to `toBeAttached` but the underlying timing sensitivity may recur

Investigate with more generous `waitFor` patterns or explicit state assertions before the interaction step.

Affected files: `tests/js/e2e/forms.spec.js`, `tests/js/e2e/tooltip-hovercard.spec.js`
Effort: 1 hour

### copyButton and kebabButton convention

These two primitives were reclassified as NOT_COMPONENTS (simple element factories) rather than migrated to the `createX(opts) -> {el, methods, destroy()}` convention. They return raw button elements, not `{el, ...}` instances. Architecturally defensible (they're stateless one-shot factories), but they're the only two primitives not following the unified shape.

Decision needed: leave as utilities, or wrap them for consistency.

Affected files: `assets/js/controls.js`, `assets/js/index.js`, `tests/js/unit/api-convention.test.js`
Effort: small if wrapping, zero if leaving
